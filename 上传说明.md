# 上传到 GitHub 说明

本文件夹（`GitHub/`）是为「开源发布到 GitHub」准备的配套文件：

- `README.md` —— GitHub 仓库首页介绍（你正在看的这份）
- `LICENSE` —— MIT 开源协议
- `.gitignore` —— 排除 `build/`、`.gradle/`、`.idea/` 等无关文件

---

## 怎么把项目上传到 GitHub

**推荐方式：把整个项目仓库推上去**

1. 在 GitHub 新建一个空仓库（如 `typing-time-counter`）。
2. 把本仓库根目录全部提交（含 `Pycharm/`、`IDEA/`、`docs/` 以及本 `GitHub/` 文件夹）。
3. 把 `GitHub/README.md`、`GitHub/LICENSE`、`GitHub/.gitignore` **复制到仓库根目录**（放好后仓库首页会显示 README，且带 LICENSE 标识）。
4. 提交并推送：
   ```bash
   git add .
   git commit -m "首次开源：码字计时器 v0.5"
   git push
   ```

> 也可以直接以本 `GitHub/` 文件夹内容作为仓库根，但那样源码就不在仓库里了，不推荐。

## GitHub Release 挂什么文件

在仓库的 **Releases → Draft a new release** 里，附加两个构建产物 zip（需先本地重新构建，见下）：

| 文件 | 给谁用 |
| --- | --- |
| `IDEA/build/distributions/typing-time-counter-idea-0.5.zip` | IntelliJ IDEA 用户 |
| `Pycharm/build/distributions/typing-time-counter-0.5.zip` | PyCharm 用户 |

> ⚠️ 注意：目前各工程的 `build/distributions/` 里**还没有**最新构建。
> 现有的旧 zip 是 13:41 打的，早于图标、双语介绍等改动，内容已过时。
> 上传前请务必在 IDE 里刷新 Gradle 后重新 `buildPlugin`，生成含最新图标与双语介绍的新 zip。
