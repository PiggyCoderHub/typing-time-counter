# 码字计时器 · 摸鱼休息提醒 — 需求文档（PRD）

| 项目 | 内容 |
| --- | --- |
| 文档版本 | v1.0 |
| 创建日期 | 2026-07-17 |
| 适用对象 | PyCharm 版 + IntelliJ IDEA 版（功能一致） |
| 状态 | 已实现，待本地编译验证与开源发布 |

---

## 1. 项目背景

开发者日常在 IDE（PyCharm / IntelliJ IDEA）中长期连续编码，存在两个问题：

1. **缺乏正反馈**：码字量、投入时长不可见，容易产生「今天啥也没干」的虚无感。
2. **久坐无提醒**：连续敲代码数小时不自知，忽视颈椎、视力与血液循环健康。

市面上缺少一个**既能量化码字、又能用幽默方式提醒休息、且带强打字反馈动效**的轻量插件。本项目即为此而生。

---

## 2. 目标

- **G（核心目标）**：为 PyCharm 与 IDEA 用户提供「码字计数 + 时间计数 + 定时幽默休息提醒 + 强打字动效」的一体化工具。
- **G1**：让用户的每一次敲键都有可见、有趣的正反馈。
- **G2**：用不惹人烦的幽默方式，引导用户按时起身休息。
- **G3**：同一套功能同时交付 PyCharm 与 IDEA 两个版本，体验一致。

---

## 3. 用户角色

| 角色 | 描述 | 关注点 |
| --- | --- | --- |
| 终端用户（开发者） | 在 IDE 里写代码的普通人 | 好装、好看、不卡、提醒有用不烦 |
| 维护者（开发者本人） | 插件的开发与发布者 | 代码可维护、双版本同步 |

## 4. 功能需求（Functional Requirements）

### FR1 码字计数
- FR1.1 实时统计「今日码字数」与「累计码字数」。
- FR1.2 通过编辑器文档变更事件（`DocumentListener`）全局监听，覆盖所有打开的文件与项目。
- FR1.3 粘贴、自动补全触发的文本变更同样计入（不做「仅手敲」区分）。
- FR1.4 「今日」按 IDE 本地自然日重置；「累计」长期保留。

### FR2 时间计数
- FR2.1 统计「今日活跃码字时长」（仅连续敲击期间累计）。
- FR2.2 空闲超过 **60 秒** 不计入活跃时长（去倒水 / 思考不算码字）。
- FR2.3 估算并展示实时「手速」（字符/分钟，CPM）。
- FR2.4 统计累计活跃时长。

### FR3 每日目标
- FR3.1 用户可设定每日码字目标（默认 1000，范围 100~5000）。
- FR3.2 工具窗展示目标完成进度。
- FR3.3 当日首次达成目标时触发一次祝贺（不重复打扰）。

### FR4 固定时间休息提醒
- FR4.1 累计活跃时长达到「休息间隔」时，弹出右下角气球通知（BALLOON）。
- FR4.2 提醒文案为幽默风格（多条文案随机）。
- FR4.3 通知带两个动作按钮：
  - 「现在就去走走 🚶」：确认休息，重置休息计时。
  - 「我再战一会儿 ⚔️」：贪睡，按「贪睡时长」延后再次提醒。
- FR4.4 提供总开关，可关闭提醒仅保留计数。

### FR5 强打字动效（核心差异化）
- FR5.1 **码字舞台**：每敲一个字符生成带光晕的彩色粒子，自底部上飘并淡出。
- FR5.2 **流动弹幕带**：底部横向滚动展示近期输入字符流。
- FR5.3 **炸裂大字**：每满 100 字或达成当日目标，中央炸出祝贺大字。
- FR5.4 背景为渐变 + 漂浮光点，整体富有活力。
- FR5.5 动画以约 30 帧/秒（Timer ≈ 33ms）本地绘制，不得明显占用主线程导致卡顿。

### FR6 休息倒计时可视化
- FR6.1 工具窗底部展示渐变弧形进度环（Countdown Ring）。
- FR6.2 环上显示「距离下次休息剩余时间」。
- FR6.3 端点发光；被「贪睡」或「确认休息」时进度归零重算。

### FR7 设置项
- FR7.1 入口：`Settings → 码字计时 & 休息提醒`。
- FR7.2 字段：开启提醒（布尔）、休息间隔（5~180 分）、贪睡时长（1~60 分）、每日目标（100~5000 字）。
- FR7.3 修改即时生效，无需重启。

### FR8 数据持久化
- FR8.1 使用 `PersistentStateComponent` + `@State/@Storage` 持久化到本地 XML。
- FR8.2 数据包含：总字数、今日字数、各时长、最后敲键时间戳、设置项、累计休息计时等。
- FR8.3 数据仅存本地，**不上传、不联网**。

### FR9 幽默文案体系
- FR9.1 文案集中管理于 `FunMessages`，分类：状态（STATUS）、里程碑（MILESTONE）、达标（GOAL）、提醒（REMIND）、贪睡（SNOOZE）、关闭（DISMISS）、空闲（IDLE）。
- FR9.2 状态栏文案每秒轮换，提升趣味。

## 5. 非功能需求（Non-Functional Requirements）

| 编号 | 类别 | 要求 |
| --- | --- | --- |
| NFR1 | 性能 | 动画与计数不得导致编辑器输入延迟；CPU 占用常规可忽略 |
| NFR2 | 兼容性 | 声明兼容 IntelliJ 平台 2021.3+（sinceBuild 213），兼容区间 213~263.*（2021.3~2026.2） |
| NFR3 | 一致性 | PyCharm 版与 IDEA 版功能、UI、文案完全一致 |
| NFR4 | 零依赖 | 不引入任何第三方运行时库，仅用 JDK + IntelliJ 平台 API |
| NFR5 | 隐私 | 所有数据本地存储，无网络请求 |
| NFR6 | 可维护 | 源码结构清晰，动效 / 逻辑 / 文案分层，双版本同源 |

## 6. 技术约束与实现要点

- **平台**：IntelliJ Platform Plugin（Java + Gradle）。
- **语言/工具链**：Java 11，Gradle + `org.jetbrains.intellij` 1.17.3。
- **构建目标**：PyCharm 版 `intellij.type = "PC"`；IDEA 版 `type = "IC"`。
- **核心模块**（包名 `com.coder.typingtime`）：
  - `TypingCounterService`：计数 + 休息逻辑 + 持久化（核心）。
  - `TypingStage` / `CountdownRing`：强动效绘制。
  - `CounterPanel` / `TypingToolWindowFactory`：工具窗装配。
  - `BreakSettingsConfigurable`：设置页。
  - `FunMessages`：文案。
  - `InitActivity`：启动拉起服务。
- **全局监听**：`EditorFactory.getInstance().eventMulticaster.addDocumentListener`，避免逐编辑器注册。
- **并发**：打字监听器用 `CopyOnWriteArrayList` 管理，规避并发修改。

---

## 7. 双插件策略

- 两份插件**源码一致**，仅 `plugin.xml`（id / 名称）与 `build.gradle.kts`（`type`）不同：
  - PyCharm 版 id：`com.coder.typing-time-counter`
  - IDEA 版 id：`com.coder.typing-time-counter-idea`
- 因两者均仅依赖平台模块（`com.intellij.modules.platform`），**理论可合并为单一插件同时兼容两边**；当前按用户要求保持双文件夹、双 id 独立交付。

## 8. 验收标准（Acceptance Criteria）

- [x] 在 PyCharm / IDEA 中打开对应文件夹可成功 `buildPlugin` 并产出 zip。
- [x] 从磁盘安装后，工具窗出现且实时统计正确。
- [x] 敲字时舞台出现上飘光点、弹幕与炸裂大字，无明显卡顿。
- [x] 达到休息间隔弹出幽默气泡，两个动作按钮行为正确。
- [x] 设置项修改即时生效；数据跨项目累计、按日重置。
- [x] 关闭提醒后仅计数不弹窗。
- [x] 数据仅存本地，无网络请求。

## 9. 范围外（Out of Scope）
